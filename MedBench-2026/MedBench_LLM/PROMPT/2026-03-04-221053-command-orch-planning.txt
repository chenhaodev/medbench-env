  ---
  Design Section 2 of 4 — Model Clients & Voting Logic

  Principle: Claude as Default, DeepSeek as Challenger

  Claude is trusted by default. DeepSeek runs are cheap "challengers" — they can
   only override Claude if they unanimously agree on a different answer.

  Tier 1 — Claude Opus 4.6 + DeepSeek-R1 ×3

  1. Run Claude Opus 4.6  → answer_claude
  2. Run DeepSeek-R1 ×3  → [a1, a2, a3]
  3. If all 3 DS agree AND differ from Claude → use DS majority answer
  4. Otherwise → use Claude's answer

  Claude wins in ~80% of cases. DeepSeek only overrides when it has unanimous
  conviction — which is a strong signal Claude may have hallucinated or misread
  the question.

  Tier 2 — Claude Sonnet 4.6 + DeepSeek-V3 ×2

  1. Run Claude Sonnet 4.6  → answer_claude
  2. Run DeepSeek-V3 ×2    → [a1, a2]
  3. If both DS agree AND differ from Claude → use DS answer
  4. Otherwise → use Claude's answer

  Dawid-Skene (future cycles)

  After cycle 1 submission feedback, per-task model accuracy is known.
  Dawid-Skene replaces the hard unanimous-override rule with a learned 
  reliability weight per model per task type. This is why we save raw_votes/ —
  it seeds the estimator for cycle 2.
